# LLM 모델 비교 및 선정 보고

## 1-1. 개요

본 보고서에서는 Gemini 3.1 flash, Claude Sonnet 4.6, GPT 5.5 Instant 위 세 가지 모델을 활용하여 실험을 진행하였다. 사용 환경은 전부 동일하게 웹으로 통일하였으며, 무료 플랜에서 기본으로 동작하는 보급형 모델을 선정하였다.

## 1-2. 평가 기준 및 결과

### 평가 지표

**1. 문맥 및 의도 보존**
- 사용자가 입력한 원문의 뉘앙스와 핵심 메시지를 왜곡하지 않았는지 측정한다.
- 측정 방법: 원문의 요구사항이 번역본에 누락 없이 반영되었는지 확인

**2. 톤앤매너 적합성**
- 사용자가 지정한 수신자와의 관계에 맞는 적절한 비즈니스 프레이징과 어휘를 선택했는지 측정한다.
- 측정 방법: 말투가 딱딱한 직역 투인지, 상황에 맞는 유연하고 캐주얼한 표현인지 구분해서 평가

**3. 문화적 맥락**
- 단순히 문법적으로 정확한지를 넘어 해당 문화권에서 실제로 사용하는 표현인지, 환각은 없는지 측정한다.
- 측정 방법: 영어권 국가에서 실제로 자주 쓰는 이디엄이나 이메일 에티켓을 적재적소에 썼는지 확인

**4. 출력 형식 준수 여부**
- 프롬프트에 요구한 출력 규격을 깨뜨리지 않고 출력했는지 확인한다.
- 측정 방법: 제시한 구조(Final Draft / Key Expressions / Cultural Tip)대로 출력 포맷을 지켰는지 여부 확인

**5. 속도**
- 단순 속도 측면에서 어느 모델이 빠르게 답변을 내놓았는지 확인

**6. 페르소나 지침 반영**
- 사용자가 규정한 페르소나의 성격, 역할에 맞는 말투를 구사하였는지 확인

## 1-3. 모델별 결과 요약

세 모델 모두 10턴의 대화를 수행하며 큰 환각 현상을 보이는 일은 없었다.

- **Gemini 모델**의 경우 포맷을 지키며 순서 표기하는 데 오류가 나거나 캐주얼한 톤을 유지해 달라는 요청을 무시하고 정중하고 단호한 최종안을 내놓는 등의 문제가 있었지만, 문화적 맥락과 사용자에게 전달하는 팁, 논리 부문에서는 정확했다.

- **Claude 모델**의 경우 속도는 압도적으로 느렸지만 실제로 대학생이 쓸 법한 표현과 문어체를 섞어 사용했으며 출력 형식을 완벽히 준수했다. 하지만 문화적 팁을 제시하는 부분에서 전달을 모호하게 해 페르소나 지침에 어긋나는 모습을 보였다.

- **GPT 모델**의 경우 빠른 속도로 페르소나 지침에 맞게 직설적 어투로 핵심만을 전달했으며, 다방면으로 프롬프트의 지침을 완벽히 수행했고 논리나 사실관계에도 오류가 없어 GPT 모델을 최종 모델로 선정했다.

### 평가 점수 비교

## 1-4. 최종 선택 모델 선정과 그 이유

**최종 선정 모델: GPT 5.5 Instant**

세 모델 모두 10턴의 문맥을 유지하는 데 무리는 없었지만, 클로드의 경우 다소 부자연스럽고 격의 없는 한국어 문장 구사, 페르소나 성격 지침을 이행하지 않았으며 가장 느린 속도로 답을 출력하는 양상을 보였고, 제미나이의 경우 출력 형식은 준수했으나 내용의 가독성과 정렬 부문에서 타 모델에 비해 상대적으로 떨어지는 모습을 보였다. 지피티 모델의 경우 10턴을 진행하는 동안 특별히 지침을 어기는 모습이나 환각 현상을 보이지 않았고, 전반적인 프롬프트 규정을 잘 준수하여 최종 모델로 선정하였다.
